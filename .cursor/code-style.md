#Code style

- use Tab for indentation

##Comments for functions

- Public function is called outside of a module (or is mean to call outside of a module)
- Doesn't write a detailed comments to not public functions
- Instead a detailed comment prefer a clear signature of function
- Doesn't write a details of implementations in comments to a function
- Write in in comment detail of input or output function which you can't to describe at a function signature
